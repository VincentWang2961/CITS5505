Story of the project

This project is my first web development project. Before this semester, I tried to learn web development through online resources like public courses such as Harvard's CS50 and Wisconsin–Madison's CS571. However, I found it difficult to learn frameworks like React without first understanding how to work with the native DOM. I didn't fully grasp the fundamentals of DOM manipulation or how it functions, so I stopped these courses and switched to learning through documentation instead.

My main focus became JavaScript after a friend who is a front-end development expert advised me that JavaScript is the most crucial part of front-end development. Although I had some JavaScript knowledge and experiences from doing labs of previous public courses, I felt overwhelmed when starting the project because it's completely nothing at the beginning and I wasn't sure about the proper workflow. To solve this, I studied some blog posts and personal portfolio websites to understand typical webpage components. I also used generative AI like Claude-3.7 and ChatGPT-o1 to help design the CSS styles since my own designs were ugly and it's a big work.

Starting with a basic HTML structure and CSS with help of AI, I attempted to implement interactive effects using JavaScript. During this phase, I also used AI to debug code, but discovered its limitations - even when I described the errors, the AI couldn't always fix them. Two solutions worked best: either using the browser's developer console to identify exactly where the error occurred (then feeding that information to the AI), or deleting problematic JavaScript code entirely and asking the AI to rewrite it. Some persistent issues arose because some AI-generated CSS occasionally conflicted with other elements - these were simple to fix once found, but hard to detect, especially since AI tools often miss CSS-related issues.

After completing the main development, I researched industry best practices to improve the code quality and added proper source citations. For the resume section, being a first-year student with no work experience, I creatively substituted my own information with pictures of my pet rat and created a fictional rat-themed resume, which turned out to be both functional and humorous.

Regarding the web design, I employed full-color blocking on the homepage combined with a typing animation for the title display - a common pattern seen in various blogs. Originally considered using background images, I ultimately opted for gradient colors to maintain visual consistency across the page. The design features two color schemes supporting different modes: light and dark modes. The layout utilizes floating boxes that trigger hover animations when scrolled into view. Third-party icon libraries are integrated to visually represent my technical stack and enhance button functionality, achieving both aesthetic appeal and usability.

The website features a pill-shaped floating sidebar for page navigation and dark mode toggling, adopting a modern minimalist style. Below the sidebar resides a "Back to Top" button that maintains consistent styling and positioning with the sidebar. Notably, the framework intentionally avoids Bootstrap despite my prior coursework experience with it, instead using media queries for responsive design practice and performance optimization. jQuery was excluded due to its declining relevance in modern workflows, replaced by native JavaScript implementations. For API requests, the contemporary Fetch API is preferred over traditional AJAX. CSS and JavaScript files are modularly organized by functionality to facilitate maintenance and reuse.

During initial development, numerous bugs emerged but were systematically resolved through continuous debugging. While time constraints prevented exhaustive testing, the current build demonstrates stable operation across major browsers, having passed cross-browser compatibility checks. The codebase maintains clear documentation and standardized citation practices for third-party resources.
